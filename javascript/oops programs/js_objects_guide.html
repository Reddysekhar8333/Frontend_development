Part 1 — Chapter 1

Creating objects using object literals { }, dot access, add/update/delete

Python mapping: Similar to dict, but JS objects can also carry methods.
let person = {
  name: "Shekhar",
  age: 25
};

console.log(person.name);   // Shekhar
person.city = "Hyderabad";  // add
person.age = 26;            // update
delete person.city;         // delete
Part 2 — Chapter 2

Square brackets, function vs method, shorthand properties

Access with square brackets

  • Use when key has spaces/special chars or is in a variable.
const key = "full name";
let user = { "full name": "Shekhar Reddy" };
console.log(user[key]);      // Shekhar Reddy

Function vs Method

function greet(){ console.log("Hello"); } // function

let user2 = {
  name: "Shekhar",
  greet: function(){ console.log("Hello " + this.name); } // method (uses this)
};
user2.greet();

Shorthand properties

let name = "Shekhar", age = 25;
let person1 = { name, age }; // same as { name: name, age: age }
Part 3 — Chapter 3

Copy by value vs copy by reference

Primitives (number, string, boolean, null, undefined, bigint, symbol) copy by value. Objects/arrays copy by reference. 
let a = { x: 1 };
let b = a;  // reference copy
b.x = 5;
console.log(a.x); // 5 (original affected)
Part 4 — Chapter 4

Object.assign() and spread operator

Both are shallow copies (nested objects still shared).
let base = { a: 1, nest: { b: 2 } };

let copy1 = Object.assign({}, base);
let copy2 = { ...base };

copy1.nest.b = 9;
console.log(base.nest.b); // 9 (shared reference!)
Part 5 — Chapter 5

Deep cloning / structuredClone

let o1 = { a: { b: 2 }, arr: [1,2,3] };
let o2 = structuredClone(o1);
o2.a.b = 7;
o2.arr.push(4);

console.log(o1.a.b);   // 2 (separate)
console.log(o1.arr);   // [1,2,3] (separate)

Python analogy: copy.deepcopy().

Object.keys(), Object.values(), Object.entries()

Object.keys(person);   // ["name", "age"]
Object.values(person); // ["Shekhar", 25]

Part 8 — Chapter 8

Property flags + freeze/seal/preventExtensions

Property flags

let obj = { name: "Shekhar" };
Object.defineProperty(obj, "name", {
  writable: false,      // can't change value
  enumerable: true,     // shows up in loops
  configurable: false   // can't redefine/delete
});
obj.name = "Reddy";     // no effect in strict mode throws

Freeze / Seal / Prevent Extensions

let x = { a: 1 };
Object.freeze(x); // no add/update/delete

let y = { a: 1 };
Object.seal(y);   // can update existing, no add/delete

let z = { a: 1 };
Object.preventExtensions(z); // can't add new props
Part 9 — Chapter 9

Prototype & prototypal inheritance

function Person(name){ this.name = name; }
Person.prototype.greet = function(){
  console.log("Hi " + this.name);
};

const p = new Person("Shekhar");
p.greet(); // "Hi Shekhar"

Python analogy: class inheritance; in JS each object links to a prototype.

Part 10 — Chapter 10

Symbols as keys

const id = Symbol("id");
let user = { [id]: 123, name: "Shekhar" };
console.log(Object.keys(user)); // ["name"] — symbol key is hidden from keys()

Symbols create guaranteed-unique property keys.

Part 11 — Chapter 11

Property existence tests

let person = { name: "Shekhar" };
"name" in person;               // true  (checks prototype chain too)
person.hasOwnProperty("name");  // true  (own property only)

Python: 'name' in d for dicts.

JS Trap: extracting a method loses this.
class Person {
  constructor(name){ this.name = name; }
  greet(){ console.log("Hello, I am", this.name); }
}
const p = new Person("Shekhar");
const f = p.greet;
f();                 // undefined (this lost)
f.bind(p)();         // works
